ERPNext 3PL Demo: краткие сценарии для проверки

Клиентский портал

Логин: alpha.client@example.test

Пароль: AlphaClient2026!

Ссылка: https://erpnext.77.237.244.169.sslip.io/client/receiving-notice/list

1. Создание Receiving Notice

Клиент заранее сообщает складу о входящей поставке.

Что работает:

  • клиент открывает раздел Receiving Notices;
  • создает новое уведомление о поставке;
  • указывает ожидаемые товары и количества в описании;
  • документ автоматически привязан к Demo Client Alpha;
  • клиент не видит и не может использовать данные других клиентов.

2. Просмотр своих Receiving Notices

Клиент видит только свои уведомления о поставках.

Демо-записи:

  • ASN-ALPHA-001
  • ASN-ALPHA-002
  • ASN-ALPHA-003

3. Просмотр остатков

Клиент видит свой складской остаток.

Демо-SKU:

  • ALPHA-001
  • ALPHA-002
  • ALPHA-003

4. Создание Shipment Request

Клиент может создать запрос на отгрузку товара со склада.

Что работает:

  • клиент указывает reference, дату, адрес назначения и описание товаров;
  • складская команда видит этот запрос в ERPNext.

Демо-записи:

  • SHIP-ALPHA-001
  • SHIP-ALPHA-002

5. Инструкции по расхождениям

Клиент может оставить инструкцию по проблемам приемки.

Примеры:

  • принять недостачу;
  • оставить поврежденный товар на удержании;
  • запросить дополнительную проверку.

Менеджер склада

Логин: warehouse.manager@example.test

Пароль: WarehouseManager2026!

Ссылка: https://erpnext.77.237.244.169.sslip.io/desk/3pl-warehouse

1. Работа со складской панелью

Менеджер работает через workspace 3PL Warehouse.

Доступны:

  • Stock Entries;
  • отчет ASN vs Received;
  • отчет Receiving Discrepancies;
  • Pick Lists;
  • складские отчеты.

2. Приемка товара

Товар принимается через стандартный ERPNext stock flow.

Что работает:

  • приемка выполняется через Stock Entry;
  • товар сначала попадает в Temporary Receiving - 3;
  • товар не размещается сразу в финальные storage locations.

3. Проверка расхождений

Менеджер может смотреть расхождения между ожидаемым и фактически принятым товаром.

Поддерживаются типы расхождений:

  • отсутствующий товар;
  • неожиданный товар;
  • разница в количестве;
  • поврежденный товар;
  • проблема качества.

4. Putaway

После проверки товар можно переместить из receiving area в storage location через стандартное складское перемещение ERPNext.

5. Outbound flow

Менеджер может работать с исходящими процессами.

Доступны:

  • Pick Lists;
  • shipment requests, созданные клиентом.

6. Отчеты

Менеджер может открывать операционные отчеты:

  • ASN vs Received;
  • Receiving Discrepancies;
  • Containers Report;
  • Stock Balance;
  • Stock Ledger.

Краткая логика решения

  • Используется чистая установка ERPNext v16.
  • 3PL-функциональность добавлена поверх стандартного ERPNext.
  • Core ERPNext stock logic не изменялся.
  • Клиенты работают через клиентский портал.
  • Складские пользователи работают через ERPNext Desk.
  • Владение товаром моделируется как Client + SKU.
  • Коробки/контейнеры моделируются как Three PL Container.
Базовый receiving flow: Receiving Notice → Receiving Area → Verification → Putaway → Storage Location
